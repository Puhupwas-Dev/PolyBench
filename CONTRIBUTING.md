# Internal Development Notes

This document is for personal reference only. This repository is a fork of an open‑source project, adapted for private evaluation workflows.

## Making Changes

- Work against the latest local copy of the `main` branch.
- Keep changes focused on the specific modifications needed (e.g., configuration, documentation, or evaluation scripts).
- Run local tests before committing to ensure nothing is broken.
- Use clear commit messages describing what was changed and why.

## Issues and Discussions

Since this is a personal/internal repository, issues and pull requests are not monitored. All changes are made directly by the repository owner.
